# Partner Landing Page Template - Field Requirements

## Overview
Every approved strategic partner automatically gets a modern landing page created using the `PublicPCRLandingV2` component. This document outlines all required and optional fields for automatic landing page generation.

## Component Location
**File**: `/tpe-front-end/src/components/reports/PublicPCRLandingV2.tsx`
**Demo URL**: `http://localhost:3002/demo/pcr-v2` (local) or `https://tpx.power100.io/demo/pcr-v2` (production)

## Database Table
**Table**: `strategic_partners`
**Total Columns**: 151 fields

---

## Required Fields for Landing Page

### 1. Hero Section
| Field | Type | Usage | Fallback |
|-------|------|-------|----------|
| `company_name` | VARCHAR | Partner name in hero | 'Partner' |
| `tagline` | TEXT | Subtitle under partner name | Empty string |
| `logo_url` | VARCHAR | Logo for swarm effect + summary section | '/power100-logo.png' |
| `final_pcr_score` | NUMERIC | PowerConfidence score display | 99 |

### 2. Focus Areas Section
| Field | Type | Usage | Fallback |
|-------|------|-------|----------|
| `focus_areas_served` | TEXT (JSON array) | Display growth areas with icons | Section hidden if empty |

**Valid Focus Area Values**:
- `greenfield_growth` → Map icon
- `closing_higher_percentage` → Target icon
- `controlling_lead_flow` → Users icon
- `installation_quality` → Award icon
- `hiring_sales_leadership` → UserPlus icon
- `marketing_automation` → Zap icon
- `customer_retention` → Heart icon
- `operational_efficiency` → Settings icon
- `technology_integration` → Cpu icon
- `financial_management` → DollarSign icon

### 3. Summary Section
| Field | Type | Usage | Fallback |
|-------|------|-------|----------|
| `ai_summary_title` | TEXT | Italicized section title | Hidden if empty |
| `ai_extended_summary` | TEXT | Main summary content | Falls back to `value_proposition` |
| `value_proposition` | TEXT | Fallback summary content | Section hidden if both empty |
| `logo_url` | VARCHAR | Partner logo display (left column) | '/power100-logo.png' |

### 4. Key Metrics Section
| Field | Type | Usage | Fallback |
|-------|------|-------|----------|
| `key_metrics` | JSONB | Results cards with metrics | Section hidden if empty |

**Key Metrics Structure**:
```json
[
  {
    "metric": "+12%",
    "label": "Average Revenue Growth"
  },
  {
    "metric": "89%",
    "label": "Client Retention Rate"
  }
]
```

### 5. Testimonials Section
| Field | Type | Usage | Fallback |
|-------|------|-------|----------|
| `client_testimonials` | JSONB | Customer quotes and info | Section hidden if empty |

**Testimonial Structure**:
```json
[
  {
    "quote": "This partner transformed our business...",
    "name": "John Smith",
    "company": "ABC Contractors",
    "role": "CEO"
  }
]
```

### 6. CTA Buttons
| Field | Type | Usage | Fallback |
|-------|------|-------|----------|
| `cta_primary_text` | VARCHAR | Primary button text | 'Schedule Introduction' |
| `cta_secondary_text` | VARCHAR | Secondary button text | 'Download Report' |

## Automated Landing Page Generation Flow

### 1. Partner Approval Trigger
```sql
UPDATE strategic_partners
SET status = 'approved', is_active = true
WHERE id = [partner_id];
```

### 2. Required Data Validation
Before generating landing page, verify:
- ✅ `company_name` is not null
- ✅ `logo_url` is not null
- ✅ `focus_areas_served` has at least 1 area
- ✅ `final_pcr_score` is not null
- ✅ Either `ai_extended_summary` OR `value_proposition` exists

### 3. AI Enhancement (If Missing)
If AI fields are empty, trigger AI processing:
```javascript
// Pseudo-code
if (!partner.ai_extended_summary) {
  await processPartnerWithAI(partner.id);
}
```

### 4. URL Generation
Landing page automatically available at:
- **Development**: `http://localhost:3002/partner/{partner_id}/pcr`
- **Production**: `https://tpx.power100.io/partner/{partner_id}/pcr`

Or using subdomain (if configured):
- `https://{power100_subdomain}.tpx.power100.io`

## Field Priority Levels

### Priority 1 (Critical - Page Won't Work Without These)
1. `company_name`
2. `logo_url`
3. `final_pcr_score`
4. `focus_areas_served`

### Priority 2 (Important - Significant Content Loss)
5. `ai_extended_summary` or `value_proposition`
6. `tagline`
7. `key_metrics`
8. `client_testimonials`

### Priority 3 (Nice to Have - Enhanced Experience)
9. `ai_summary_title`
10. `landing_page_videos`
11. `cta_primary_text` / `cta_secondary_text`
12. `contact_email` / `contact_phone`

## Maintenance and Updates

### When to Regenerate Landing Pages
- Partner logo changes
- PowerConfidence score updates (quarterly)
- Focus areas change
- New testimonials added
- Key metrics updated

### Automated Updates
Landing pages automatically reflect changes when:
- Database fields are updated
- Partner profile is edited in admin dashboard
- AI processing runs and updates summaries

### Manual Review Needed
Human review recommended when:
- New partner onboarding complete
- Major profile changes
- Negative feedback received
- PowerConfidence score drops significantly
